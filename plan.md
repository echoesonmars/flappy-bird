### 0. Техническая база и структура проекта

- **Движок:** `Python 3.x + pygame`.
- **Цель:** оффлайн-игра с возможностью онлайн-запросов к OpenAI (через отдельный модуль).
- **Целевые платформы:** Windows (в первую очередь), возможность сборки в `.exe` (через `pyinstaller`/`nuitka` в будущем).
- **Структура папок:** см. `structure.md` (`assets/`, `src/`, `data/`).
- **Главные модули в `src/`:**
  - `main.py` — входная точка, цикл игры, машина состояний экранов.
  - `settings.py` — константы (разрешение, FPS, цвета, физика, пути к ресурсам).
  - `player.py` — логика игрока (физика, анимация, столкновения).
  - `obstacles.py` — генерация труб/барьеров, логика движения и подсчёт очков.
  - `ui.py` — отрисовка и логика кнопок/меню/экранов.
  - `ai_core.py` — работа с OpenAI API, очереди запросов, потоки.
  - `data_manager.py` — сохранение/загрузка прогресса и настроек.

### 1. Пользовательский интерфейс (UI) и экраны

Навигация построена на **машине состояний**. Каждый экран — отдельный класс с методами:
- `handle_events(events)` — обработка ввода.
- `update(dt)` — логика.
- `draw(surface)` — отрисовка.

#### 1.1. Главный экран (Main Menu)

- **Фон:**
  - Анимированный параллакс-фон (несколько слоёв спрайтов).
  - Двигающаяся "земля"/платформа как в Flappy Bird.
- **Лого и стиль:**
  - Логотип с неоновым свечением / пульсацией.
  - Общий вайб — техно-мистицизм (древние глифы + футуристичная UI-сетка).
- **Элементы:**
  - Кнопка `[ИГРАТЬ]` — переход в игровой экран.
  - `[ГАРДЕРОБ]` — экран скинов.
  - `[ОРАКУЛ]` — экран AI-чата.
  - `[НАСТРОЙКИ]` — настройки звука и API-ключа.
  - Отображение **лучшего счета** и текущей **валюты**.

#### 1.2. Экран Настроек (Settings)

- Ползунки/кнопки:
  - Громкость **музыки**.
  - Громкость **эффектов (SFX)**.
- Поле ввода:
  - `OpenAI API Key` (можно спрятать/показать символы, хранится локально, не пушится в git).
- Кнопки:
  - `"Сохранить"` — записывает настройки в файл (например, `settings.json` в `data/`).
  - `"Очистить прогресс"` — сброс сейва (с подтверждением).

#### 1.3. Экран "Гардероб" (Skins & Customization)

- Карусель скинов:
  - Отображение разблокированных/заблокированных скинов.
  - Цена в валюте за разблокировку (если есть внутренняя экономика).
  - Превью анимации выбранного скина.
- Фича **"Загрузить свой спрайт"**:
  - Кнопка, которая вызывает `tkinter.filedialog` с фильтром `*.png`.
  - Игра загружает PNG, масштабирует под нужный размер.
  - Создает маску коллизий через `pygame.mask.from_surface`.
  - Пользовательский скин сохраняется в `data/` (путь и метаданные — в `save_data.json`.

#### 1.4. Экран "Оракул" (AI Chat)

- Стиль:
  - Терминал, голографический монолит или "камень с рунами", через который говорит ИИ.
- Элементы:
  - Область истории сообщений (скролл).
  - Поле ввода текста.
  - Кнопка `"Спросить"`.
  - Индикация того, что ИИ "печатает" (анимация точек / бегущая строка).
- Экономика:
  - Каждый вопрос стоит **N единиц валюты**.
  - Кнопка неактивна, если валюты не хватает.

#### 1.5. Экран Game Over

- Показывает:
  - Текущий результат.
  - Лучший результат.
  - Заработанную за попытку валюту.
  - Фразу от ИИ (контекстный комментарий).
- Кнопки:
  - `"Ещё раз"` — рестарт раунда.
  - `"В меню"` — возврат в Main Menu.

### 2. Основной геймплей (Core Loop)

Ключевой приоритет — **отзывчивость** и **60 FPS**. Никаких фризов из-за сети: все запросы к ИИ идут отдельно.

#### 2.1. Игрок (Player)

- Физика:
  - Каждый кадр к вертикальной скорости добавляется `GRAVITY`.
  - При нажатии `SPACE` / клике — скорость по Y устанавливается в отрицательное значение (`JUMP_FORCE`).
  - Позиция обновляется с учётом `dt` (delta time).
- Визуал:
  - Наклон спрайта в зависимости от скорости:
    - При прыжке — нос вверх.
    - При падении — плавный поворот вниз.
  - Анимация крыльев/двигателей по таймеру кадров.
- Столкновения:
  - Маски столкновений с трубами и землей.
  - Детект коллизий через `pygame.mask.overlap`.

#### 2.2. Препятствия (Obstacles)

- Генератор:
  - Периодический спавн пар "верхняя труба + нижняя труба".
  - Случайное смещение по вертикали в допустимом диапазоне.
  - Динамический размер зазора: чем дальше летишь, тем у́же проход.
- Логика движения:
  - Трубы двигаются влево с постоянной или растущей скоростью.
  - Удаление труб, вылетевших за левый край экрана.
- Очки:
  - Между трубами — **триггер-зона** (невидимый прямоугольник).
  - При прохождении игрока через триггер:
    - +1 к счету.
    - Звук "coin" / "data shard".

#### 2.3. Валюта (Осколки данных / Искры)

- Спавн:
  - Периодически появляются собираемые объекты, иногда в рискованных позициях.
  - Можно привязать спавн к прогрессу (после каждых N пройденных труб).
- Механика:
  - При сборе:
    - Звук приятного "дзинь".
    - Частицы (искры, пиксели).
    - Увеличение баланса валюты.
- Использование:
  - Разблокировка скинов.
  - Оплата вопросов Оракулу.

#### 2.4. Сложность и баланс

- Параметры:
  - Скорость движения труб.
  - Размер зазора.
  - Частота появления валюты.
  - Высота случайного смещения труб.
- Базовый план:
  - Стартовая сложность комфортная.
  - Каждые N очков — небольшое увеличение сложности.

### 3. Визуал и атмосфера (Сеттинг)

Визуал выдержан в стиле "техно-мистицизм".

- Спрайты:
  - Вместо птицы — дрон/сущность из света.
  - Вместо труб — руины с глифами, голографические барьеры или столбы данных.
- Анимации:
  - Sprite sheets для крыльев/двигателей.
  - Частицы при прыжке, столкновении, сборе валюты.
- Параллакс:
  - Минимум 3 слоя фона (дальний космос/руны, средние структуры, ближние элементы).
  - Разная скорость движения слоёв создаёт глубину.

### 4. Интеграция OpenAI (Модуль интеллекта)

ИИ — это "Высший Разум", который комментирует полеты, даёт советы и может менять параметры игры.

#### 4.1. Техническая реализация

- Отдельный модуль `ai_core.py`:
  - Функции: `send_prompt_async(...)`, `get_last_response()`, очереди запросов.
  - Использование `threading` или `concurrent.futures` для асинхронных вызовов.
  - Без блокировок основного игрового цикла.
- Конфигурация:
  - API-ключ берётся из `.env` или пользовательских настроек.
  - Возможность полностью **отключить ИИ**, если ключ не задан (игра работает как обычный Flappy Bird).

#### 4.2. Контекстный Game Over

- При смерти формируется скрытый промпт с данными:
  - `score`, `high_score`, `coins_earned`, `reason` (столкнулся с трубой / упал / задел землю).
- ИИ возвращает короткую фразу/абзац:
  - В духе: *"Ты преодолел 15 барьеров, но пал от собственной жадности..."*.
  - Текст выводится с эффектом печатной машинки.

#### 4.3. Чат-бот (Оракул)

- Экран чата использует `ai_core.py` для отправки сообщений.
- Возможности:
  - Подсказки по игре.
  - Мистические фразы, расширяющие лор.
  - Реакция на поведение игрока (частые смерти, высокий счёт и т.п.).

#### 4.4. Динамическая сложность (продвинутая фича)

- ИИ может возвращать JSON с параметрами:
  - Например: `{ "gravity_multiplier": 1.1, "gap_size_offset": -10 }`.
- Игра при следующем запуске раунда подстраивает настройки:
  - Злобный/доброжелательный ИИ влияет на геймплей.

### 5. Звуковой дизайн

Используется `pygame.mixer`.

- Музыка:
  - Спокойный эмбиент в меню.
  - Более динамичный трек во время полёта или усиленная версия меню.
  - Гладкие переходы (кроссфейд или постепенное изменение громкости).
- Эффекты (SFX):
  - Короткий звук прыжка.
  - "Дзинь" при прохождении труб и сборе валюты.
  - Удар/краш при смерти.
  - Звук "техно-печати" при выводе текста от ИИ.
- Громкость всех звуков регулируется через настройки.

### 6. Сохранение данных (Система прогрессии)

Файл `save_data.json` лежит в `data/` (или в системной папке пользователя). Через `data_manager.py` реализуются:

- Структура JSON (базовый вариант):
```json
{
  "high_score": 42,
  "currency": 150,
  "unlocked_skins": ["default_drone", "neon_angel"],
  "equipped_skin": "neon_angel",
  "ai_chat_history": [
    { "role": "user", "content": "..." },
    { "role": "assistant", "content": "..." }
  ],
  "settings": {
    "music_volume": 0.8,
    "sfx_volume": 0.9
  }
}
```

- Моменты автосохранения:
  - При окончании раунда (Game Over).
  - При покупке/смене скина.
  - При изменении настроек.
  - При завершении диалога с Оракулом (опционально).

### 7. Контролы и UX-мелочи

- Управление:
  - PC: `SPACE` / ЛКМ / тап (для будущей мобильной версии).
  - `ESC` — пауза или выход в меню (зависит от экрана).
- Подсказки:
  - На стартовом экране игры — текст "Нажми SPACE, чтобы взлететь".
  - Ненавязчивые подсказки от ИИ, если игрок часто умирает в одном и том же месте.

### 8. Roadmap (этапы разработки)

1. **MVP-геймплей:**
   - Игрок, трубы, столкновения, очки.
2. **Визуал и звук базового уровня:**
   - Параллакс-фон, базовые спрайты, базовые SFX.
3. **Система прогресса и скинов:**
   - Сейвы, валюта, гардероб без кастомных PNG.
4. **Интеграция OpenAI:**
   - Контекстные сообщения на Game Over, базовый Оракул.
5. **Продвинутая косметика и кастомные спрайты:**
   - Загрузка своих PNG, улучшенные частицы.
6. **Динамическая сложность и дополнительные фишки:**
   - Реакция ИИ на поведение игрока, тюнинг сложности.
