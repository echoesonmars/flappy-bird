### assets/

Тут лежат все медиа-ресурсы игры: графика, звуки и шрифты. Они не содержат логики — только данные, которые ты подгружаешь из кода.

---

### assets/sprites/

**Общее:** спрайты, которые используются для отрисовки фона, земли, птиц и UI. Обычно их грузишь через `pygame.image.load` и при необходимости масштабируешь.

- **0.png – 9.png**
  - **Что это:** цифры от 0 до 9.
  - **Где использовать:** рендер счета/рекорда в HUD.
  - **Как использовать в коде:**
    - Загружаешь все 10 файлов в список/словарь: индекс соответствует цифре.
    - Для вывода числа разбиваешь его на символы и для каждой цифры выводишь соответствующий спрайт.

- **background-day.png / background-night.png**
  - **Что это:** фоны дневной и ночной сцены.
  - **Где использовать:** задний план игрового экрана и главного меню.
  - **Как использовать:**
    - Повторяешь (тайлишь) текстуру по горизонтали.
    - Двигаешь её влево, создавая эффект бесконечного фона (параллакс-слой дальнего плана).

- **base.png**
  - **Что это:** нижняя "земля"/платформа, по которой визуально скользит камера.
  - **Где использовать:** передний план внизу экрана, поверх фона, под птицей.
  - **Как использовать:**
    - Реализуешь скроллинг как у фона, но с чуть другой скоростью (для эффекта глубины).
    - Также можно использовать как хитбокс "земли" (столкновение при падении).

- **bluebird-*.png / redbird-*.png / yellowbird-*.png**
  - **Что это:** анимационные кадры трёх скинов птицы (синяя, красная, жёлтая) в трех фазах взмаха: `downflap`, `midflap`, `upflap`.
  - **Где использовать:** основной спрайт игрока в зависимости от выбранного скина.
  - **Как использовать:**
    - Для каждого цвета создаешь список кадров `[down, mid, up]`.
    - В `Player` по таймеру кадра циклически переключаешь индекс анимации.
    - Вращаешь спрайт (через `pygame.transform.rotate`) в зависимости от вертикальной скорости.

- **pipe-green.png / pipe-red.png**
  - **Что это:** спрайты труб двух цветов (обычно зелёные — стандарт, красные — вариация / повышенная сложность / декоративный элемент).
  - **Где использовать:** препятствия в игровом поле.
  - **Как использовать:**
    - Один и тот же спрайт можно переворачивать по вертикали для верхней/нижней трубы (`pygame.transform.flip`).
    - Цвет можно использовать как код сложности, типа красные трубы появляются на более высоком уровне.

- **gameover.png**
  - **Что это:** надпись "Game Over".
  - **Где использовать:** экран смерти/проигрыша.
  - **Как использовать:**
    - Отрисовывать поверх затемненной сцены, по центру экрана.
    - Можно добавить лёгкую анимацию появления (scale-in / fade-in).

- **message.png**
  - **Что это:** стартовое сообщение/подсказка (обычно "Get Ready!" с кнопкой "Tap").
  - **Где использовать:** перед началом раунда, когда птица стоит на месте.
  - **Как использовать:**
    - Показать поверх сцены, пока игрок не сделал первый "взмах" (нажал SPACE/кликнул).

### assets/audio/

**Общее:** звуки событий в игре. Грузи через `pygame.mixer.Sound` или `pygame.mixer.music` (для музыки, если добавишь позже).

- **die.ogg / die.wav**
  - **Что это:** звук смерти.
  - **Когда проигрывать:** при столкновении с трубой/землёй или при выходе за границы.

- **hit.ogg / hit.wav**
  - **Что это:** звук удара/столкновения.
  - **Когда проигрывать:** в момент коллизии с препятствием или землей (можно сочетать с `die` или слегка смещать по времени).

- **point.ogg / point.wav**
  - **Что это:** звук получения очка.
  - **Когда проигрывать:** когда игрок проходит через зазор между трубами и срабатывает триггер увеличения счета.

- **swoosh.ogg / swoosh.wav**
  - **Что это:** звук "свиста"/перелистывания.
  - **Когда проигрывать:**
    - При переходе между экранами (меню → игра, игра → Game Over).
    - При открытии/закрытии окон UI (настройки, гардероб).

- **wing.ogg / wing.wav**
  - **Что это:** основной звук взмаха/прыжка.
  - **Когда проигрывать:** каждый раз, когда игрок нажимает `SPACE`/кликает, и птица делает "подтягивание" вверх.

> **Примечание:** для каждого события выбери один формат (ogg или wav) и используй его в коде; второй можно оставить как резерв или удалить позже, если станет не нужен.

### assets/fonts/

- **Geist-VariableFont_wght.ttf**
  - **Что это:** современный variable-шрифт Geist.
  - **Где использовать:**
    - Тексты в меню (названия кнопок, логотип, подписи).
    - HUD (счёт, надписи вроде "PAUSED", "GAME OVER", если не используешь спрайтовые цифры).
  - **Как использовать в коде:**
    - Загрузить через `pygame.font.Font("assets/fonts/Geist-VariableFont_wght.ttf", size)`.
    - Делать разные размеры для заголовков/обычного текста.
  - **Дизайн-настройка:**
    - Под выбранный визуальный стиль (техно-мистицизм) можно использовать полужирные/капсовые надписи, неоновые цвета и легкий внешний `outline` через несколько отрисовок шрифта с разным цветом/смещением.

### Как в целом работать с assets из кода

- **Пути:** зашить базовые пути в `settings.py`, например:
  - `SPRITES_DIR = "assets/sprites"`
  - `AUDIO_DIR = "assets/audio"`
  - `FONTS_DIR = "assets/fonts"`
- **Лоадеры ресурсов:** сделать маленький модуль/класс `AssetLoader`, который:
  - При старте игры грузит все необходимые спрайты/звуки/шрифты.
  - Складывает их в словари по ключам (`"background_day"`, `"pipe_green"`, `"wing_sfx"` и т.п.).
  - Даёт удобные функции типа `get_sprite("player_blue")`, `get_sound("point")`, `get_font("ui_large")`.

Так `assets.md` превращается в живую документацию: смотришь в файл — сразу понятно, что за ресурс, где его использовать и как к нему обратиться из кода.
